🏃 HAR App — Clasificador de Actividades Humanas

Aplicación de escritorio con Tkinter + ConvNeXt-Base para reconocimiento de acciones en imágenes

Descripción general

Aplicación de escritorio con interfaz gráfica que clasifica imágenes de personas en 15 actividades humanas utilizando un modelo ConvNeXt-Base entrenado con transfer learning. El pipeline de preprocesamiento replica exactamente el usado durante el entrenamiento.

Modelo utilizado

ConvNeXt-Base ~88M parámetros 80.26% accuracy (TTA)

Pipeline de preprocesamiento

La app replica el mismo pipeline que data_adecuate.py:

1
Lectura — cv2.imread() en formato BGR
2
Resize — a 288×384 (W×H) con interpolación adaptativa: INTER_CUBIC (upscale) o INTER_AREA (downscale)
3
CLAHE — Conversión a LAB → ecualización del canal L (clipLimit=2.0, tileGridSize=8×8) → reconversión a BGR
4
Conversión a RGB — para compatibilidad con el modelo PyTorch
5
Normalización ImageNet — mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]

Estructura del código

preprocess_image(img_path)

Lee una imagen, aplica resize + CLAHE y devuelve una tupla (img_clahe_rgb, img_original_rgb). La imagen original se usa para mostrar en la GUI; la procesada con CLAHE se usa para la clasificación.

build_model()

Construye ConvNeXt-Base con head custom, carga los pesos .pth limpiando prefijos (_orig_mod. de torch.compile(), module. de DataParallel/SWA) y pone en modo evaluación.

classify(model, img_rgb)

Transformada → tensor → forward pass → softmax → devuelve las 15 clases ordenadas por probabilidad descendente.

HARApp — Interfaz gráfica

Layout de dos paneles con paleta warm neutral (fondo claro, acentos violeta/verde):

ComponenteDescripción
HeaderTítulo + nombre del modelo y archivo de pesos
ToolbarBotón "Seleccionar imagen" + nombre del archivo seleccionado
Panel izquierdoImagen original (sin procesar), escalada proporcionalmente (máx 340px alto) + tarjeta con predicción y confianza
Panel derecho15 ttk.Progressbar con porcentaje real (0–100%) para cada clase, ordenadas por probabilidad
Status barMensajes de estado: modelo cargado, procesando, resultado

Flujo de uso

  1. Ejecutar python app/app.py
  2. Esperar a que cargue el modelo (status bar indica "Listo")
  3. Clic en "📂 Seleccionar imagen"
  4. Elegir una imagen (.jpg, .png, .bmp, .webp)
  5. Se muestra la imagen original a la izquierda
  6. A la derecha aparecen las 15 barras de progreso con el porcentaje real
  7. La clase predicha se muestra en grande con su emoji y porcentaje

Notas técnicas

  • El modelo se carga una sola vez al iniciar y permanece en memoria
  • Inferencia en CPU (map_location="cpu") — no requiere GPU
  • Las barras son ttk.Progressbar con escala absoluta (0–100%), mostrando el porcentaje real de cada clase
  • La GUI muestra la imagen original sin procesar; el preprocesamiento (CLAHE) se usa internamente para la clasificación
  • El pipeline es idéntico al del entrenamiento, garantizando consistencia
